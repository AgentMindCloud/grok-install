# Migrating from v2.13 to v2.14

**TL;DR — there is nothing to migrate.** v2.14 is **purely additive**. Every
valid v2.13 `grok-install.yaml` is a valid v2.14 file, unchanged. The
`version` field accepts both `"2.13"` and `"2.14"` under v2.14 tooling.

---

## What changed

Only one new block — entirely optional:

| Block | Required? | Summary |
|---|---|---|
| `visuals` | No | Preview card, install flow, demo media, dashboard, theme, accessibility |

No existing field was renamed, removed, or made stricter. The JSON Schema's
`additionalProperties: false` still applies, so unknown keys continue to
fail validation — the same behaviour as v2.13.

---

## Adopting v2.14

You have three options.

### Option 1 — do nothing

Leave your file at `version: "2.13"`. It will pass both the v2.13 schema and
the v2.14 schema. The gallery will render it using the default dark-premium
surface.

### Option 2 — bump the version only

Change one line:

```diff
- version: "2.13"
+ version: "2.14"
```

Everything else stays the same. The gallery still falls back to defaults for
any unset visual field.

### Option 3 — adopt the visuals block

Append a `visuals` block to opt into branded rendering. Minimal:

```yaml
visuals:
  accent_color: "#00F0FF"
  preview_card:
    style: "futuristic"
  demo_media:
    auto_generate: true
```

Full field reference: [`docs/v2.14/visuals.md`](../v2.14/visuals.md).
Flagship example: [`examples/janvisuals/grok-install.yaml`](../../examples/janvisuals/grok-install.yaml).

---

## One-liner migration

Once `grok-install-cli` ships v2.14 support you can run:

```bash
grok-install migrate --from 2.13 --to 2.14
```

The command only rewrites the `version` field. It never touches any other
block. Passing `--with-visuals` will also scaffold a default `visuals`
block you can then customise.

---

## Validation

Validate against the v2.14 schema (Draft 2020-12):

```bash
ajv validate \
  -s schemas/v2.14/schema.json \
  -d grok-install.yaml \
  --spec=draft2020 -c ajv-formats
```

The v2.13 schema is still published at
`schemas/grok-install-v2.13.schema.json` and will continue to validate
v2.13 files indefinitely.

---

## Compatibility matrix

| Your file says | v2.13 schema | v2.14 schema |
|---|---|---|
| `version: "2.13"` | passes | passes |
| `version: "2.14"`, no `visuals` block | fails (unknown version) | passes |
| `version: "2.14"` + `visuals` block | fails (unknown key) | passes |

Point old tooling at the v2.13 schema; new tooling at the v2.14 schema.
Nothing else has to change.
