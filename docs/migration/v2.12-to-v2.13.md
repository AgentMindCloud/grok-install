# Migration Guide: v2.12 → v2.13

---

## Overview

v2.13 is a **minor breaking change**. The only required action for most agents
is expanding tool declarations from name-only to full schema form.

**v2.12 support timeline:** v2.12 is supported until v3.0 (target: ~6 months
from v2.13 release). CI will warn on v2.12 files starting at v2.14.

---

## What Breaks

### 1. Tool declarations (the only hard break)

v2.12 allowed tool names as bare strings or minimal objects:

```yaml
# v2.12 — no longer valid in v2.13
tools:
  - post_thread
  - reply_to_mention
```

v2.13 requires full JSON Schema parameter and return type definitions:

```yaml
# v2.13 — required
tools:
  - name: "post_thread"
    description: "Posts a thread to X after explicit user approval"
    parameters:
      type: "object"
      properties:
        tweets:
          type: "array"
          items: {type: "string", maxLength: 280}
          minItems: 1
          maxItems: 25
        require_approval:
          type: "boolean"
          default: true
      required: ["tweets"]
    returns:
      type: "object"
      properties:
        thread_url: {type: "string"}
        posted_at: {type: "string", format: "date-time"}
```

### 2. `version` field

Change `version: "2.12"` to `version: "2.13"`.

---

## What Is New (Additive — No Action Required)

These blocks are optional. Agents that omit them continue to work unchanged.

| Block | Default behaviour when absent |
|---|---|
| `structured_output` | Responses are untyped (v2.12 behaviour) |
| `parallel_tool_calls` | `false` — sequential execution (v2.12 behaviour) |
| `tool_choice` | `"auto"` (v2.12 behaviour) |
| `rate_limits` | No per-tool limiting (v2.12 behaviour) |
| `cost_limits` | No cost limits (v2.12 behaviour) |
| `telemetry` | `enabled: false` — no data sent |

## Side-by-Side Examples

### Minimal agent — before and after

**v2.12:**

```yaml
---
version: "2.12"
name: "Reply Bot"
description: "Replies to X mentions"
llm:
  provider: "xai"
  model: "grok-4"
  api_key_env: "GROK_API_KEY"
tools:
  - reply_to_mention
```

**v2.13:**

```yaml
---
version: "2.13"
name: "Reply Bot"
description: "Replies to X mentions"
llm:
  provider: "xai"
  model: "grok-4"
  api_key_env: "GROK_API_KEY"
tools:
  - name: "reply_to_mention"
    description: "Replies to a mention after reading context"
    parameters:
      type: "object"
      properties:
        mention_id: {type: "string"}
        reply_text: {type: "string", maxLength: 280}
        require_approval: {type: "boolean", default: false}
      required: ["mention_id", "reply_text"]
    returns:
      type: "object"
      properties:
        reply_url: {type: "string"}
        posted_at: {type: "string", format: "date-time"}
```

## Automated Migration

Once `grok-install-cli` ships, run:

```bash
pip install grok-install
grok-install migrate --from 2.12 --to 2.13
```

The tool will:
1. Detect bare tool names and wrap them in minimal schema stubs
2. Update the `version` field
3. Print a diff and ask for confirmation before writing

Until the CLI is available, use the [live validator](https://agentmindcloud.github.io/grok-install)
to check your file against the v2.13 JSON Schema.

---

## Checklist

- [ ] Change `version: "2.12"` to `version: "2.13"`
- [ ] Expand all tool declarations to include `name`, `description`, `parameters`, `returns`
- [ ] (Optional) Add `rate_limits` for any X-posting tools
- [ ] (Optional) Add `cost_limits` if running on shared infrastructure
- [ ] Validate with `ajv validate -s schemas/grok-install-v2.13.schema.json -d grok-install.yaml`
