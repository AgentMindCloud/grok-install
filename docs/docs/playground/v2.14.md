## The three examples

### 1. JanVisuals — reference

The canonical v2.14 reference. Demonstrates every feature: responsive
`srcset`, declared contrast ratio, sidecar transcript, and an
**Install on X** CTA. [Load →](?sample=janvisuals) ·
[full YAML](../v2.14/visuals.md#janvisuals-full-example)

### 2. Research swarm card

A `layout: banner` example designed for wide hero placements. Shows the
three-agent research flow (researcher → critic → publisher) with a
single "Open in playground" CTA. [Load →](?sample=visuals-research-swarm)

### 3. Reply bot card

A compact `layout: inline` example for sidebars and lists. The schema
accepts a terse block — only `type`, `src`, and `alt` are strictly
required. [Load →](?sample=visuals-reply-bot)

## What the preview renders

- `type: image` → `<img>` with `loading="lazy"`, `alt=""` from YAML.
- `type: video` → `<video controls preload="metadata" poster="…">`.
- `type: gif` → same as image.
- `type: carousel` → a labelled placeholder (the carousel runtime lives
  in the install-card renderer on X).
- `accessibility.reduced_motion` → honoured by the browser's
  `prefers-reduced-motion: reduce` media query via CSS.
- `cta.{label,url}` → a 44 × 44 px button styled with
  `var(--grok-cyan)` for AA contrast.

## Accessibility

- The preview card is a `<figure>` with `<figcaption>` — screen readers
  announce the media type correctly without ARIA.
- `alt` is schema-required: editing it out fails validation.
- Contrast is driven by CSS variables (`--grok-cyan`, `--grok-text`,
  `--grok-panel`) — toggle the site theme (top-right) to verify.
- Full checklist: [WCAG 2.2 AA →](../v2.14/accessibility.md).
